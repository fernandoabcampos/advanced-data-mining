1.1.1 Random forest simple (1 punto)¶

La idea bsica del bagging es utilizar el conjunto de entrenamiento original para generar centenares o miles de conjuntos similares usando muestreo con reemplazo. En este concepto est basado el algoritmo Random Forest, la combinacin de varios rboles de decisin, cada uno entrenado con una realizacin diferente de los datos. La decisin final del clasificador combinado (la Random Forest) se toma por mayora, dando el mismo peso a todas las decisiones parciales tomadas por los clasificadores base (los rboles).

from sklearn.ensemble import RandomForestClassifier

def random_forest_cross_val_score( X, Y, n_estimators = 100):
    clf = RandomForestClassifier(n_estimators = 100)
    model = clf.fit(X, Y)

    precision = cross_val_score(model, X, Y, scoring = 'precision_macro', cv = 5)
    print('Precisin (Macro): \t%.4f' % np.mean(precision))
    return precision, model

1.1.2 Out-of-bag (1 punto)¶

Una ventaja del bagging usado en el Random Forest es que cada uno de los rboles de decisin ha sido entrenado con una combinacin diferente de los datos (muestreo con reemplazo), o sea que cada uno de los rboles no ha visto una determinada parte de los datos originales. Esto define una especie de conjunto de test para cada uno de los rboles, llamado out-of-bag, que puede ser usado para estimar el error del modelo sin necesidad de usar el conjunto de test real que creamos previamente, ni de usar estrategias de cross-validation.

Implementacin: Usando los conjuntos *X_train_pca* e *y_train*, entrenar un modelo Random Forest con 100 rboles de decisin. Mostrar la precisin de este modelo en el *out-of-bag* y en el conjunto *X_test_pca*.
Sugerencia: usar el mdulo *RandomForestClassifier* de sklearn. Para aprender ms sobre *out-of-bag* y sobre como usar este mdulo (incluyendo el atributo *oob_score_*), os recomendamos los siguientes enlaces:
http://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestClassifier.html
http://scikit-learn.org/stable/auto_examples/ensemble/plot_ensemble_oob.html

Anlisis: La precisin obtenida en el *out-of-bag* y en el conjunto de test son comparables? Era de esperar?

Son comparables (y tras ejecutar algunas veces, los resultados suelen ser bastante parecidos en todas las ejecuciones).

Segn define [1] Albon, C. (2018) en el Out-of-bag para cada observacin, el algoritmo de aprendizaje compara el verdadero valor de la observacin con la prediccin de un subconjunto de rboles que no estn entrenados para usar esa observacin. La puntuacin general se calcula y proporciona una medida nica del rendimiento del RandomForest. Con esto, la estimacin de la puntuacin OOB es una alternativa ms rpida a la validacin cruzada y por lo tanto era de se esperar que tuviera una precisin adecuada y comparable con la prediccin.

1.1.3 Probabilidad por clase (1 punto)¶

Otra ventaja del bagging usado en el Random Forest es que cada uno de los rboles de decisin, entrenado con una combinacin diferente de los datos, puede obtener un resultado diferente. En los problemas de clasificacin, el resultado de cada rbol se considera como un voto diferente, y la prediccin final del modelo es la clase que haya obtenido ms votos teniendo en cuenta todos los rboles.

Estos votos individuales de los rboles tambin se pueden usar para estimar la probabilidad con la que el modelo prev cada una de las clases, siendo la probabilidad para cada clase igual al nmero de votos obtenidos para aquella clase dividido entre el nmero de rboles.

2. Combinación secuencial de clasificadores base diferentes¶

2.1 Stacking (1 punto)¶

Un clasificador de stacking usa como atributos las predicciones hechas por otros clasificadores en lugar de los datos originales de entrada.

Para construir nuestro clasificador de stacking vamos a usar las predicciones hechas en el conjunto de test por los clasificadores:

  • utilizados en los ejercicios anteriores en la PEC 4
  • los utilizados en la PEC 3 (K-Nearest neighbors Classifier (knc), Support Vector Machines Classifier (svmc) y Neural Network Classifier (nnc))
  • Discriminant Analysis (dac)

los dos ltimos os los damos en archivos adjuntos. Estas predicciones se pueden cargar con el siguiente cdigo:
